# 🦖 Rex Design Document

Version: 0.1 (Hatchling)

---

# Why Rex Exists

Programming should be simple, enjoyable, and powerful.

Rex is designed to remove unnecessary complexity while remaining fast enough for professional software development.

---

# Our Mission

Create a programming language that:

- Is easy to learn
- Is enjoyable to write
- Is powerful enough for real-world software
- Produces clear and readable code
- Helps developers instead of fighting them

---

# Motto

Simple. Fast. Powerful.

---

# Core Principles

## 1. Readability First

Code is read more often than it is written.

Every feature should improve readability.

---

## 2. Simplicity Over Complexity

If two solutions are equally powerful, choose the simpler one.

---

## 3. Consistency

One clear way to solve a problem is better than many confusing ways.

---

## 4. Helpful Errors

Error messages should explain what happened and how to fix it.

---

## 5. Performance

Programs should run efficiently without making the language difficult to learn.

---

## 6. Community

Rex is an open programming language built for everyone.

Ideas are welcomed, but every feature must improve the language.

---

# The Rex Golden Rule

Every new feature must make Rex:

- Simpler
- Faster
- Safer

If it doesn't improve at least one of these, it doesn't belong in Rex.
